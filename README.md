# Art RAG — технический отчет о разработке RAG-ассистента по искусству

## 1. Цель проекта
Разработать ассистента для ответов на вопросы об искусстве, художниках и направлениях на основе принципа **Retrieval-Augmented Generation (RAG)**. Система должна использовать реальные данные из Википедии, векторный поиск и генеративную модель, при этом задача **не решается простым промптингом** — ответ всегда опирается на найденный контекст.


### 2. Ключевые компоненты
- **Источник данных**: датасет `legacy-datasets/wikipedia` (20220301.en) с Hugging Face.  
- **Предобработка**: фильтрация статей по ключевым словам об искусстве, разбивка на чанки по 500 символов (перекрытие не используется).  
- **Векторный индекс**: FAISS (индекс плоского косинусного расстояния), модель эмбеддингов `all-MiniLM-L6-v2`.  
- **Генерация**: Mistral AI API (модель `mistral-small-latest`), промпт ограничивает ответ только контекстом.  
- **Интерфейс**: Streamlit‑приложение с полем ввода и отображением источников.

## 3. Этапы разработки (чекпоинты)

### Чекпоинт 1: Сбор и подготовка данных
- Загружено 100 000 статей из дампа Wikipedia.
- Фильтрация по ключевым словам: `painting`, `artist`, `renaissance`, `cubism` и т.д.
- Получено **1243 статьи**, разбитых на **8762 чанка**.
- Данные сохранены в `data/raw/` и `data/processed/`.

### Чекпоинт 2: Построение индекса
- Выбрана модель эмбеддингов `all-MiniLM-L6-v2` (баланс скорость/качество).
- Рассчитаны эмбеддинги для всех чанков, построен индекс FAISS.
- Индекс и метаданные сохранены в `data/index/`.

### Чекпоинт 3: RAG‑пайплайн и интерфейс
- Реализованы классы `Retriever`, `Generator` (через Mistral API) и `RAGPipeline`.
- Создан веб‑интерфейс на Streamlit с отображением источников.
- Ручное тестирование подтвердило релевантность ответов.

### Чекпоинт 4: Оценка качества и деплой
- Сформирован тестовый набор из 30 вопросов с эталонными ответами.
- Проведена оценка по метрикам ROUGE.
- Проект задеплоен на Hugging Face Spaces (доступ по [ссылке]).
- Настроен CI с линтерами (flake8, black, isort).

## 5. Инструкция по запуску

1. Клонировать репозиторий:  
   `git clone https://github.com/your-repo/art-rag.git`
2. Создать и активировать виртуальное окружение:  
   `python -m venv venv` → `source venv/bin/activate` (Linux) / `venv\Scripts\activate` (Windows)
3. Установить зависимости:  
   `pip install -r requirements.txt`
4. Создать файл `.env` с ключом Mistral:  
   `MISTRAL_API_KEY=your_key_here`
5. Загрузить и подготовить данные:  
   `python scripts/fetch_wikipedia_art_sample.py 100000`  
   `python scripts/build_index.py`
6. Запустить интерфейс:  
   `streamlit run app/streamlit_app.py`

## 7. Использованные технологии

- **Datasets**: Hugging Face `datasets`, `pyarrow`
- **Векторизация**: `sentence-transformers`, `faiss-cpu`
- **RAG-фреймворк**: `langchain`, `langchain-mistralai`
- **API‑клиент**: `mistralai` (0.1.8), `httpx`
- **Интерфейс**: `streamlit`
- **Тестирование**: `pytest`, `rouge-score`

## 8. Выводы и перспективы

- Разработан fully‑functional RAG‑ассистент, который корректно отвечает на вопросы об искусстве, опираясь на проверенные источники.
- Продемонстрировано, что задача не сводится к простому промптингу — без этапа поиска ответы были бы невозможны.
